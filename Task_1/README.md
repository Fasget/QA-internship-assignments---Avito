# Тестовое задание для стажировки QA-инженера

## Задание 1: Поиск багов на странице выдачи

Ниже перечислены обнаруженные проблемы на скриншоте страницы с результатами поиска по бренду «Author», а также указаны их приоритеты (high, medium, low) с учётом влияния на функциональность и пользовательский опыт.


### 1. Нерелевантные объявления в выдаче по бренду «Author» (**High**)
- **Описание:** Несмотря на выбранный фильтр «Author», в поисковой выдаче отображаются велосипеды других брендов (например, «Горный велосипед 26» без указания бренда и «Велосипед Atom 24ds»).
- **Причина:** Некорректная работа фильтра по бренду или неправильная классификация со стороны продавцов. 


### 2. Отображение объявлений из других регионов (**High**)
- **Описание:** В списке, который должен содержать объявления сначала из Москвы, а потом только из других регионов встречаются велосипеды из Липецка, раньше чем закончились все обьявления из Москвы.
- **Причина:** Ошибка в механизме сортировки/фильтрации по региону.


### 3. Некорректная сортировка по стоимости (**Medium**)
- **Описание:** При выборе сортировки «Сначала дороже» объявления отображаться не в том порядке, который ожидает пользователь.
- **Причина:** Ошибка в логике сортировки по стоимости.


### 4. Несоответствие выбраной категории товара (**Medium**)
- **Описание:** пользователь может искать именно горные велосипеды Author, но видит модели, которые позиционируются к другому типу (MTB, Sport).
- **Причина:** Неверная категория, указанная продавцом, или некорректная привязка объявления к выбранному типу велосипеда.


### 5. Несоответствие общего количества объявлений (**Low**)
- **Описание:** В пагинации отображаются номера страниц до 100, но при заявленных 61 объявлениях не может быть 100 страниц.
- **Причина:** Возможная ошибка в подсчёте общего количества объявлений или некорректная работа пагинации.
