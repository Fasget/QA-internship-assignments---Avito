# Список обнаруженных багов

**Окружение** Windows 11 x64, Python 3.11.9, pytest 8.3.4, requests 2.32.0

| №  | 1 |
|----|---|
| **Заголовок**           | Некорректный код ответа при пропущенных обязательных полях |
| **Тест-кейс**           | №3 (Создание объявления без обязательных полей) |
| **Приоритет и серьёзность** | High, critical |
| **Стенд**               | `POST https://qa-internship.avito.com/api/1/item` |
| **Предусловия**         | Сервис запущен и доступен по указанному адресу |
| **Шаги**                | 1. Отправить POST-запрос без обязательных полей (например, без `name` и `price`).<br><br>**Пример запроса:**<br>POST /api/1/item<br>Content-Type: application/json<br>{<br>  "sellerID": 1234345231<br>}<br><br>2. Просмотреть статус-код и тело ответа. |
| **Ожидаемый результат** | Код ответа: `400 Bad Request`.<br>Тело ответа содержит поля `{"result": {...}, "status": "<string>"}`, указывающие на ошибку валидации. |
| **Фактический результат** | Возвращается код `200` и сообщение о «Сохранении объявления», хотя необходимые поля не переданы.<br><br>Пример тела ответа:<br>```json<br>{ "status": "Сохранили объявление - dde78c80-e02f-4212-a602-8c6de81a7c0b" }``` |
