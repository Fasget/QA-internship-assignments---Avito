## Создание объявления (POST /api/1/item)

### 1: Успешное создание объявления

**Предварительные условия:**
1. Сервис доступен по адресу: `https://qa-internship.avito.com/api/1/item`.
2. Допустим, мы используем валидный `sellerID` в теле запроса.

**Шаги:**
1. Отправить `POST`-запрос на `/api/1/item` с телом (JSON), где присутствуют все необходимые поля:
   ```json
   {
     "sellerID": 1234345231,
     "name": "dsds",
     "price": 1,
     "statistics": {
       "contacts": 3,
       "likes": 123,
       "viewCount": 12
     }
   }
   ```
2. Проверить код ответа (ожидается `200`).
3. В ответе ожидать JSON вида:
   ```json
   {
     "status": "<string>"
   }
   ```
   Например: `"status": "Сохранили объявление - <UUID>"`.
4. Убедиться, что в поле `status` содержится корректное сообщение.

**Ожидаемый результат:**
- Статус-код `200`.
- Тело ответа содержит поле `"status"` (тип `string`), без ошибок.

---

### 2: Создание объявления с некорректным sellerID

**Предварительные условия:**
1. Сервис доступен по адресу: `https://qa-internship.avito.com/api/1/item`.

**Шаги:**
1. Отправить `POST`-запрос на `/api/1/item`, где `sellerID` некорректен (например, строка `"abc123"` или значение вне ожидаемого диапазона).
   ```json
   {
     "sellerID": "abc123",
     "name": "TestInvalidSeller",
     "price": 100
   }
   ```
2. Проверить код ответа — ожидается `400`.
3. Проверить, что в теле ответа есть поля `"result"` и `"status"`, отражающие ошибку.

**Ожидаемый результат:**
- Статус-код `400`.
- Тело ответа:
  ```json
  {
    "result": {},
    "status": "<string>"
  }
  ```
  с описанием неверного `sellerID`.

---

### 3: Создание объявления без обязательных полей

**Шаги:**
1. Отправить `POST`-запрос на `/api/1/item`, где отсутствуют, к примеру, `name` и `price`:
   ```json
   {
     "sellerID": 1234345231
   }
   ```
2. Проверить код ответа — ожидается `400`.
3. Убедиться, что в теле ответа присутствуют `"result"` и `"status"`, указывающие на ошибку валидации.

**Ожидаемый результат:**
- Статус-код `400`.
- Тело ответа:
  ```json
  {
    "result": {},
    "status": "<string>"
  }
  ```
  описывающее, что отсутствуют необходимые поля.

## Получение объявления по ID (GET /api/1/item/:id)

### 4: Успешное получение объявления

**Предварительные условия:**
1. В сервисе **существует** объявление с ID, например `0cd4183f-a699-4486-83f8-b513dfde477a`.
2. Оно ранее было добавлено и доступно для чтения.

**Шаги:**
1. Отправить `GET`-запрос на `/api/1/item/0cd4183f-a699-4486-83f8-b513dfde477a`.
2. Проверить код ответа — `200`.
3. Убедиться, что в теле ответа **массив** объектов, где у первого (или каждого) присутствуют поля:
   - `id`
   - `sellerId`
   - `name`
   - `price`
   - `statistics` (как объект: `{"likes": <int>, "viewCount": <int>, "contacts": <int>}`)
   - `createdAt`

**Ожидаемый результат:**
- Статус-код `200`.
- Тело ответа, например:
  ```json
  [
    {
      "id": "<string>",
      "sellerId": "<integer>",
      "name": "<string>",
      "price": "<integer>",
      "statistics": {
        "likes": "<integer>",
        "viewCount": "<integer>",
        "contacts": "<integer>"
      },
      "createdAt": "<string>"
    }
  ]
  ```
  Допускается, что в массиве может быть несколько подобных объектов.

---

### 5: Запрос несуществующего объявления

**Шаги:**
1. Взять заведомо фейковый UUID (например, `00000000-0000-0000-0000-000000000000`).
2. Отправить `GET /api/1/item/<fake_id>`.
3. Проверить код ответа: `404`.
4. Убедиться, что тело ответа содержит поля `"result"` и `"status"` с описанием «Объявление не найдено».

**Ожидаемый результат:**
- Статус-код `404`.
- Тело ответа:
  ```json
  {
    "result": {
      "messages": {},
      "message": "<string>"
    },
    "status": "<string>"
  }
  ```
  где `message` указывает, что не найдено нужное объявление.

---

### 6: Некорректный формат ID (например, невалидный UUID)

**Шаги:**
1. Отправить `GET /api/1/item/12345` (короткая строка, не формат UUID).
2. Проверить код ответа — в зависимости от реализации, это может быть `400` или `404`.
3. Убедиться, что в теле ответа поля `"result"` и `"status"`, сигнализирующие об ошибке.

**Ожидаемый результат:**
- Статус-код `400` или `404` (уточняется спецификацией).
- Структура тела похожа на:
  ```json
  {
    "result": {
      "messages": {},
      "message": "<string>"
    },
    "status": "<string>"
  }
  ```

---

  ```json
  [
    {
      "id": "<string>",
      "sellerId": "<integer>",
      "name": "<string>",
      "price": "<integer>",
      "statistics": {
        "likes": "<integer>",
        "viewCount": "<integer>",
        "contacts": "<integer>"
      },
      "createdAt": "<string>"
    },
  ]
  ```

**Шаги:**

### 9: Получение объявлений, если у продавца их нет

**Шаги:**
1. Взять sellerID, у которого отсутствуют объявления (например, `999999999`).
2. Отправить `GET /api/1/999999999/item`.
3. Проверить код ответа — `200`.
4. Убедиться, что в теле ответа пустой массив `[]`.

**Ожидаемый результат:**
- Статус-код `200`.
- Тело ответа:
  ```json
  []
  ```
  или аналогичная структура, показывающая, что объявлений нет.
